Run GitHub Pages locally using Jekyll

Jekyll is a static site generator with built-in support for GitHub Pages and a simplified build process. Jekyll takes Markdown and HTML files and creates a complete static website based on your choice of layouts.

Prerequisites

Install Jekyll

We recommend using Bundler to install and run Jekyll. Bundler manages Ruby gem dependencies, reduces Jekyll build errors, and prevents environment-related bugs. To install Bundler:

Install Ruby. For more information, see "Installing Ruby" in the Ruby documentation.

Install Bundler. For more information, see "Bundler."

Building your site locally

  1. Open Terminal.
  2. Navigate to the publishing source for your site (your project)
  3. Download this Gemfile or uses bundle to generate a new one with bundle init
    4. # copy this content into your Gemfile
source 'https://rubygems.org'
gem "jekyll-jupyter-notebook"
gem "jekyll-paginate"
gem "jekyll-latex"
gem 'jekyll-seo-tag'
gem 'jekyll-twitter-plugin'
gem 'jekyll-theme-cayman'
gem 'jekyll-sitemap'
gem 'jekyll-feed'
gem "kramdown-parser-gfm"
gem 'jekyll-scholar', group: :jekyll_plugins
group :jekyll_plugins do
gem 'jekyll-commonmark-ghpages'
end
    Copy
  4. Run bundle install
  5. Run your Jekyll site locally with bundle exec jekyll serve
  6. To preview your site, in your web browser, navigate to http://localhost:4000
